# V1.1 Visual Calibration and Marketplace-card Visual Reranking

## Calibration workflow

V1.1 adds a real-output visual calibration gallery:

- `scripts/diagnostics/visualCalibrationGallery.ts`
- JSON report
- SVG previews
- HTML gallery

Each item records:

- structural tier
- structural score
- visual score / band
- visual breakdown
- warnings / strengths
- an empty `manualLabel` field for easy review

## What calibration showed

Two obvious V1 issues were confirmed:

1. structurally invalid layouts could still get overly generous visual scores
2. tiny-image / weak-CTA marketplace layouts were not penalized enough

The dominant remaining noisy axis is still `compositionBalance`, which currently over-flags many marketplace outputs as corner-collapsed.

## Tuning added in V1.1

- structural-status moderation on visual overall score
- stronger penalties for no-image / weak-image text-image harmony
- stronger penalties for underweighted CTA
- a vertical-stack balance adjustment to avoid over-penalizing stacked layouts

## Production reranking scope

Visual reranking remains intentionally narrow:

- `marketplace-card` only
- only among structurally non-invalid candidates
- only when effective scores are already near-equal
- visual never overrides the primary structural gate

## Current state

The production hook is now present, but on the current 300-context marketplace-card preview sample it did not yet change final winner selection.

That means V1.1 is calibrated and safely wired, but further calibration will still be needed before visual reranking becomes a large selection mover.
